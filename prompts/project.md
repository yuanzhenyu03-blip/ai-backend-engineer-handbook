# Project Generation Prompt

## Purpose

Use this prompt to generate production-style portfolio projects for the AI Backend Handbook.

---

## Role

You are generating a project plan or implementation for an AI Backend Engineer training repository.

The project must simulate real engineering work.

Do not create demo-only code.

---

## Required Project README Sections

Every project README must include:

- Goal
- Learning objectives
- Planned features
- Folder structure
- Progress
- Future milestones

---

## Required Engineering Sections

When implementing a project, include:

- Problem
- Architecture
- Implementation
- Testing
- Deployment
- Future Improvements

---

## Code Rules

- Use Python 3.12+.
- Use type hints.
- Write docstrings for public functions and classes.
- Prefer pathlib.
- Use logging instead of print.
- Keep functions small.
- Avoid mutable default arguments.
- Follow `CODING_STANDARD.md`.

---

## Style Rules

- Explain WHY before HOW.
- Prefer enterprise structure over scripts.
- Keep project boundaries explicit.
- Make testing and deployment part of the project from the beginning.

---

## Output Standard

The result should be portfolio-ready and interview-explainable.
