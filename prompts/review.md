# Review Guide

When reviewing any pull request or document:

## Repository

- Structure is consistent
- Naming is consistent
- No duplicated files

## Documentation

- Grammar
- Markdown
- Links
- TOC
- Examples

## Code

- Readability
- Type hints
- Docstrings
- Logging
- Error handling

## Lesson

- WHY before HOW
- Enterprise example
- FastAPI example
- Playwright example
- Interview questions
- Homework
- Cheat sheet

Rate:

Repository

Documentation

Code

Teaching Quality

Overall Score
