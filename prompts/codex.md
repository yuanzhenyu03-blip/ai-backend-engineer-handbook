# Lesson Rules

Every lesson must include:

- Introduction
- Learning Objectives
- Theory
- Why it exists
- How it works
- Memory Model
- Enterprise Examples
- FastAPI Example
- Playwright Example
- Common Mistakes
- Best Practices
- Interview Questions
- English Interview Expressions
- Homework
- Cheat Sheet
- Definition of Done

Lessons should explain WHY before HOW.

Lessons should be suitable for publication.

Final lessons belong under:

```text
docs/<topic>/dayXX-<topic-name>.md
```

Example:

```text
docs/python/day01-object-model.md
```

---

# Code Rules

Always:

- use type hints
- write docstrings
- prefer pathlib
- use logging instead of print
- write readable code
- keep functions small
- avoid unnecessary complexity

Code should look like production code.

---

# Documentation Rules

Documentation should be:

- clear
- professional
- consistent
- reusable

Avoid duplicated content.

Prefer improving existing files instead of creating similar files.
